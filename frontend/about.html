About the Project

Karachi Traffic Intelligence System (KTIS)

Project

KTIS is an end-to-end traffic prediction platform for Karachi, Pakistan. It combines the OpenStreetMap road network, live weather, public holidays, school and office timings, and city events to predict traffic levels (Low / Medium / High) for any road in the city. Predictions are served through a FastAPI backend, stored in Supabase (PostgreSQL + PostGIS), and visualized on an interactive Leaflet map.

Dataset

  • Road segments500,268 (OSMnx / OpenStreetMap)
  • Weather sourceOpen-Meteo hourly API
  • Context dataHolidays, schools, offices, events
  • Training rows840,000 (5,000 roads × 168 hours)
  • Features21 (road, weather, time, context)
  • StorageSupabase · PostgreSQL + PostGIS

Model

  • AlgorithmXGBoost Classifier
  • Problem typeMulti-class (Low / Medium / High)
  • Estimators200 trees, depth 8
  • Accuracy79.3% on 168,000 test rows
  • ExplainabilitySHAP feature analysis
  • SchedulerBulk predictions every 15 minutes

Pipeline

1. Ingest — OSMnx road download, Open-Meteo weather, external CSVs uploaded to Supabase  →  2. Feature engineering — time features, holiday/school/office/event flags  →  3. Training — label generation, encoding, train/test split, XGBoost  →  4. Serving — FastAPI endpoints with live feature building  →  5. Frontend — dashboard, prediction form, Leaflet traffic map.

Developers

Built as a portfolio project demonstrating full-stack ML engineering: geospatial data processing, feature engineering, model training and explainability, REST API design, scheduled inference, and interactive mapping. Stack: Python, pandas, GeoPandas, OSMnx, XGBoost, scikit-learn, SHAP, FastAPI, APScheduler, Supabase (PostgreSQL/PostGIS), Leaflet.js.
